# 验证、质量与风险设计

## 1. 文档目的

PINN 船舶水动力学仿真系统同时涉及 Web 软件、异步计算、神经网络训练、物理建模和工程结果解释。验证和质量设计需要覆盖软件正确性、物理可信度、实验可复现性和工程可用性。

## 2. 验证层级

| 层级 | 目标 | 示例 |
| --- | --- | --- |
| 单元验证 | 验证单个函数或模块行为 | 权限判断、状态机、配置转换 |
| 集成验证 | 验证模块间协作 | API 创建任务后 Worker 能读取并更新状态 |
| 引擎验证 | 验证 PhysicsNeMo 适配和训练流程 | 基准 PDE 残差下降、检查点保存 |
| 物理验证 | 验证水动力结果合理性 | 阻力系数、压力分布与参考结果对比 |
| 回归验证 | 防止版本升级破坏历史能力 | 固定基准算例指标阈值 |
| 用户验收 | 验证业务流程完整 | 从建项到报告导出完整走通 |

## 3. 软件质量策略

### 3.1 后端质量

- API 参数校验必须覆盖必填项、单位、范围和枚举值。
- 任务状态机必须有单元测试，防止非法状态转换。
- Repository 层应覆盖关键查询和事务边界。
- 权限校验应覆盖项目成员、角色、只读状态和归档状态。
- 错误码应稳定，前端不依赖后端异常文本。

### 3.2 前端质量

- 关键表单应有本地校验和后端校验结果展示。
- 任务监控页应处理运行中、成功、失败、取消和日志为空等状态。
- 结果可视化应处理大数据加载中、降采样、无结果和结果损坏。
- 多方案对比应明确单位和指标来源，避免误读。

### 3.3 Worker 与任务质量

- Worker 必须支持幂等或可恢复的关键步骤。
- 任务失败必须保存错误码、错误信息、阶段和必要上下文。
- 长任务应定期写入心跳和进度。
- 取消任务应尽量安全释放 GPU、文件句柄和临时目录。
- 检查点保存应使用临时文件完成后再原子登记。

## 4. PINN 与物理验证策略

### 4.1 基准验证算例

建议建立以下基准集合：

| 基准 | 目的 | 参考指标 |
| --- | --- | --- |
| 一维/二维解析 PDE | 验证 PINN 基础收敛 | L2 误差、残差分布 |
| 二维不可压绕流简化算例 | 验证速度压力场 | 速度剖面、压力系数 |
| Wigley Hull 或简化船型 | 验证船型阻力趋势 | 阻力系数、波形趋势 |
| 规则波传播 | 验证波动方程和自由液面 | 波高、相位误差 |
| 数据同化样例 | 验证观测数据损失 | RMSE、MAE、物理残差 |

### 4.2 误差指标

| 指标 | 说明 |
| --- | --- |
| `L2 Error` | 与解析解或参考数据的二范数误差 |
| `RMSE` | 均方根误差 |
| `MAE` | 平均绝对误差 |
| `Relative Error` | 相对误差 |
| `PDE Residual Mean` | PDE 残差均值 |
| `PDE Residual P95` | PDE 残差 95 分位数 |
| `Boundary Violation` | 边界条件违反程度 |
| `Mass Conservation Error` | 质量守恒误差 |
| `Force Coefficient Error` | 力系数误差 |

### 4.3 收敛判据

训练任务不应只看总损失下降。建议同时记录：

- 总损失。
- 各 PDE 残差损失。
- 各边界条件损失。
- 数据拟合损失。
- 验证点误差。
- 物理守恒误差。
- 梯度异常或 NaN 事件。

### 4.4 结果可信度分级

| 等级 | 描述 | 使用建议 |
| --- | --- | --- |
| A | 通过基准验证，误差低且物理约束满足良好 | 可用于方案比较和报告 |
| B | 误差可接受，但局部残差偏高 | 可用于初步分析 |
| C | 收敛不充分或边界违反明显 | 仅用于调试 |
| D | 训练失败、NaN 或结果不物理 | 不应展示为有效结果 |

系统可在结果页展示可信度标记，但应避免将 PINN 输出包装成未经验证的工程结论。

## 5. 可复现性要求

每个训练任务必须记录：

- 算例配置快照。
- 几何文件哈希。
- 数据集版本和哈希。
- PhysicsNeMo 配置。
- 网络结构和损失权重。
- 优化器和训练超参数。
- 随机种子。
- Python、PhysicsNeMo、PyTorch、CUDA 等依赖版本。
- 代码版本或构建版本。
- 运行设备信息。

## 6. 数据质量要求

### 6.1 几何数据

- 文件类型和大小合法。
- 坐标单位明确。
- 包围盒范围合理。
- STL/OBJ 网格可解析。
- 参数化几何字段完整。
- 可选检查：非流形边、重复点、法向一致性。

### 6.2 观测数据

- 字段名、单位、坐标系明确。
- 缺失值比例可报告。
- 物理量范围合理。
- 时间或空间点重复情况可报告。
- 数据版本不可被静默覆盖。

### 6.3 结果数据

- 每个结果工件有类型、单位、数据范围和生成任务。
- 大型结果文件有校验值。
- 可视化数据应记录降采样策略。

## 7. 安全与权限测试

| 场景 | 预期 |
| --- | --- |
| 未登录访问项目 API | 返回 `AUTH_REQUIRED` |
| 非项目成员访问私有项目 | 返回 `PERMISSION_DENIED` |
| 访客提交训练任务 | 返回 `PERMISSION_DENIED` |
| 项目归档后修改算例 | 返回状态错误 |
| 删除正在运行任务关联的检查点 | 禁止删除或延迟删除 |
| 访问他人对象存储文件 | 签名 URL 不应生成 |

## 8. 性能与容量设计

### 8.1 API 性能目标

| 接口类型 | 目标 |
| --- | --- |
| 普通详情接口 | P95 < 300ms |
| 列表查询接口 | P95 < 500ms |
| 文件上传初始化 | P95 < 500ms |
| 创建任务接口 | P95 < 800ms |
| 结果轻量索引 | P95 < 800ms |

### 8.2 训练任务性能观察项

- 任务等待启动时间。
- 任务运行时间。
- GPU 利用率。
- 显存峰值。
- 每秒迭代数。
- 检查点写入耗时。
- 后处理耗时。

### 8.3 大结果优化策略

- 结果文件按变量、时间步、切片分块。
- 前端默认加载预览数据。
- 对体数据生成降采样版本。
- 对曲线数据进行抽稀但保留极值点。
- 下载原始数据走异步或签名链接。

## 9. 主要风险清单

| 风险 | 严重度 | 概率 | 影响 | 缓解措施 |
| --- | --- | --- | --- | --- |
| PINN 对复杂船体三维湍流问题难以收敛 | 高 | 高 | 结果不可用 | 从简化物理和基准算例开始，建立可信度分级 |
| 用户误把未验证结果作为工程结论 | 高 | 中 | 决策风险 | 报告中明确模型假设、误差和可信度 |
| PhysicsNeMo 版本变化导致适配失效 | 中 | 中 | 训练任务失败 | 引擎适配层、锁定依赖、基准回归测试 |
| GPU 资源不足 | 中 | 高 | 任务等待或失败 | 任务优先级、资源配额、CPU/GPU Worker 分离 |
| 大体积结果导致前端卡顿 | 中 | 高 | 使用体验差 | 分块、降采样、服务端预处理 |
| 几何数据质量差 | 中 | 中 | 训练域错误 | 几何预处理、校验报告和错误提示 |
| 配置过于灵活导致难以校验 | 中 | 中 | 任务失败率高 | JSON Schema、模板化配置、分步校验 |
| 实验不可复现 | 高 | 中 | 科研可信度下降 | 强制配置快照和依赖版本记录 |
| 权限隔离缺陷 | 高 | 低 | 数据泄露 | 权限测试、审计日志、对象存储访问控制 |

## 10. 测试计划建议

### 10.1 V1 原型测试范围

- 用户登录与项目权限。
- 项目、船型、几何、算例 CRUD。
- 静水阻力算例配置校验。
- PhysicsNeMo 训练任务从提交到成功完成。
- 损失曲线和基础结果展示。
- 报告生成。

### 10.2 V2 工程化测试范围

- 多用户项目协作。
- 数据集上传与数据同化。
- 检查点版本管理。
- 多方案对比。
- Worker 取消、失败、重试。
- 大结果可视化性能。

### 10.3 V3 扩展测试范围

- 新物理模板接入。
- 多 GPU 或远程执行节点。
- 更复杂三维结果。
- 与外部 CFD 或实验数据库集成。

### 10.4 V4 平台增强测试范围

- 独立消息队列或缓存组件接入。
- 实时日志推送与消息通道。
- 跨节点 Worker 调度。
- 更高并发下的任务稳定性。

## 12. 设计阶段结论

系统应优先实现一条完整、可信、可复现的最小闭环：项目创建、船型与几何、静水阻力算例、PhysicsNeMo 训练、推理后处理、结果展示和报告导出。在闭环稳定后，再扩展更复杂的水动力模型和更高维可视化能力。

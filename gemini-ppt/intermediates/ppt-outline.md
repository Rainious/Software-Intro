### Slide 2: 项目背景与行业痛点 (1分钟)
- **行业痛点**：传统船舶 CFD（计算流体力学）前处理复杂、网格划分耗时、计算资源消耗巨大、迭代周期长。
- **技术破局点**：Physics-Informed Neural Network (PINN) 将物理方程（Navier-Stokes 等）作为约束引入神经网络训练。
- **项目优势**：实现无网格化计算，支持快速参数扫描、逆问题求解和代理模型构建。

### Slide 3: 系统建设目标与边界 (1分钟)
- **核心目标**：构建一个可扩展、可追踪、可复现的 Web 化 PINN 仿真平台。
- **主要受众**：水动力研究人员、船型设计人员、教学用户。
- **系统边界**：
  - **包含**：算例管理、物理场景配置、基于 PhysicsNeMo 的训练与推理调度、结果可视化。
  - **暂不包含**：复杂 CAD 建模、完全替代高保真工业级 CFD（定位为快速评估与研究工具）。

### Slide 4: 总体架构设计 (1.5分钟)
- **解耦设计思想**：分离 Web 业务交互系统与长耗时物理训练。
- **技术栈分层**：
  - **表现层**：React (负责交互与数据可视化)
  - **API 与调度层**：Python Flask (业务流转与权限控制)
  - **任务调度层**：PostgreSQL 驱动的异步 Worker
  - **计算引擎层**：NVIDIA PhysicsNeMo (神经网络构建与物理约束求解)

### Slide 5: 核心业务流程闭环 (1分钟)
- **主线流程演示**：
  1. **资产准备**：创建船型 -> 上传/预处理几何模型。
  2. **物理配置**：定义流体参数、边界条件、PDE 约束。
  3. **模型配置**：选择网络结构 (MLP/SIREN)，设定损失权重。
  4. **异步执行**：提交训练任务 -> Worker 接管计算。
  5. **结果分析**：选择检查点推理 -> 三维流场可视化 -> 生成对比报告。

### Slide 6: 领域模块设计亮点 (1分钟)
- **模块化边界设计**：
  - **前端视图**：聚焦监控与多维度结果展示。
  - **Flask API**：聚焦鉴权、配置快照生成与任务下发（拒绝长连接阻塞请求）。
  - **Worker 层**：负责具体算力消耗（隔离 CPU 后处理任务与 GPU 训练任务）。
  - **PINN 引擎适配层**：隔离 PhysicsNeMo 框架版本迭代对业务代码的影响。

### Slide 7: 数据库与实验可追踪性设计 (1分钟)
- **科研级数据追踪**：
  - **配置快照 (Snapshot) 机制**：任务启动时锁定所有参数与代码版本，确保“历史结果不随当前配置更改而失效”。
  - **检查点管理**：追踪每次训练的最优模型与多版本模型迭代。
- **大文件外置策略**：关系型数据库仅存元数据与文件 URI，复杂的几何资产、模型权重与 VTK 流场数据接入 MinIO/S3 对象存储。

### Slide 8: 质量保证与物理验证 (1分钟)
- **双重质量保障**：
  - **软件工程层**：状态机严谨校验、基于项目的权限隔离、避免科研数据误删的软删除策略。
  - **物理科学层 (可信度分级)**：
    - 评估指标不仅监控 Total Loss，更监控 PDE 残差分布、边界条件违反情况和质量守恒误差。
    - 建立基准算例库（如简化绕流、Wigley 船型），对输出结果进行 A/B/C 可信度评级。

### Slide 9: 总结与演进路线 (0.5分钟)
- **系统核心价值**：用现代 Web 架构与深度学习重塑传统水动力学仿真工作流，提升工业设计效率。
- **未来迭代规划**：
  - **V1 (原型)**：跑通静水阻力与流场可视化核心闭环。
  - **V2 (工程化)**：多用户权限协同、多方案对比、结构化报告导出。
  - **V3 (扩展)**：规则波响应、实船数据同化接入。
  - **V4 (平台化)**：引入 Redis/K8s 支持大规模高并发计算与弹性调度。
